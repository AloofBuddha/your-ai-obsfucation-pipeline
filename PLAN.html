1. Threat Model

Adversaries we resist, in priority order. Every design decision below traces back to one of these.

#AdversaryCapabilityOur defense
A1External LLM providerSees outbound payload, logs promptsZero raw PII in payload — only typed tokens or surrogates
A2Network MITMCaptures TLS-decrypted traffic at egressSame as A1 — payload is PII-free before TLS
A3Cross-session leakageSession B receives a token derived from Session A's dataPer-session ephemeral vault key; tokens are HMAC outputs of that key
A4Vault snapshot theftAttacker exfiltrates the vault DB file at restRows encrypted with per-session key; key never written to disk
A5Audit log inspectionCompliance auditor or attacker reads the audit logLog contains tokens + types + timestamps only — no values, no reversible hashes
A6Re-identification via correlationAdversary has obfuscated doc + LLM response + side knowledgePartially mitigated; frequency/co-occurrence leakage documented as known gap
Hard guarantees (testable in CI):
  1. The outbound LLM payload contains no substring from the source document's detected entities.
  2. If a session vault is destroyed, no token from that session can be reversed by any code path.
  3. The audit log, grep'd for any original entity value, returns zero matches.

2. Key & Crypto Design

Two keys, two lifetimes — kept minimal to match PRD scope.

KeyScopeLifetimeStorage
User key One per user Persistent 32 random bytes, loaded from env var USER_KEY_<user_id> or generated on first user creation and written to .env
Session vault key (SVK) Per session Ephemeral — dies on vault.destroy() 32 random bytes, generated at session start, in-memory only, never persisted

User-key encrypts that user's documents directly (Fernet). SVK encrypts vault rows and feeds the token-derivation HMAC. Per-user isolation is enforced by key isolation: User A's key cannot decrypt User B's documents or vault.

Token construction

token   = "[{TYPE}_{SHORTID}]"           e.g.  [PHI_NAME_k7a2mqpz]
SHORTID = base32( HMAC-SHA256(SVK, TYPE || canonicalize(value)) )[:8].lower()
TYPE    = one-or-more uppercase _-separated segments, e.g. PHI_NAME,
          PHI_INSURANCE_ID, FIN_ACCOUNT_NUMBER, LEGAL_PRIVILEGE
  • 8 lowercased base32 chars (alphabet a-z2-7, RFC 4648) = 40 bits — collision probability across 10k entities in a session is negligible.
  • Deterministic within session (same SVK + same value → same token) → satisfies "same entity in same session maps to same token."
  • Non-deterministic across sessions (different SVK → different SHORTID for same value) → satisfies "irreversible across sessions" cryptographically, not just by policy.
  • HMAC one-way: token alone reveals nothing. Even with SVK, an adversary must guess the value to verify it — no offline reversal.
  • Vault row stores Fernet(SVK).encrypt(value) keyed by token. SVK dies on logout → row ciphertext is unrecoverable.
  • Grammar matches the deobfuscation regex in §3.6 (\[[A-Z]+(_[A-Z]+)+_[a-z2-7]{8}\]) — verified by a roundtrip property test.

Crypto primitives: cryptography library only — Fernet for symmetric AEAD, HMAC-SHA256 for token derivation. No KDF, no key wrapping, no hand-rolled crypto.

3.1 audit/ — append-only event log

class AuditEvent(BaseModel):
    timestamp: datetime
    session_id: str
    action: Literal["OBFUSCATE", "DEOBFUSCATE", "VAULT_CREATE",
                    "VAULT_DESTROY", "LLM_CALL"]
    entity_type: str | None     # e.g. "PHI_DIAGNOSIS"
    token_id: str | None        # the token, never the value
    metadata: dict[str, Any]    # byte counts, model name — never values

class AuditLog(Protocol):
    async def emit(self, event: AuditEvent) -> None: ...

Async writes to JSONL file (stdout in dev). Pydantic validator recursively rejects any field name (top-level or nested in metadata) containing value, original, or plaintext — closes the escape-hatch where a caller could write metadata={"original_value": "..."}. metadata values restricted to numeric/boolean/short-string scalars (no nested dicts or lists), enforced at write time.

3.2 vault/ — session-scoped token ↔ value mapping

class Vault(Protocol):
    session_id: str

    async def store(self, entity_type: str, value: str) -> str:
        """Idempotent: same value → same token within session."""

    async def lookup(self, token: str) -> str | None:
        """Returns original value or None if token not in vault."""

    async def destroy(self) -> None:
        """Wipe all session data. Subsequent lookups MUST fail."""

SQLite (aiosqlite), one row per (session_id, token), value column is Fernet ciphertext. Token-from-value derivation uses HMAC (no reverse index) — preserves the security property. Lookup direction token → value hits the SQLite index and is the path measured by the < 5ms perf test in §6.

Orphan cleanup: if the API process crashes between sessions, vault rows persist on disk but are cryptographically unreadable (SVK was in-memory only — that's by design). On startup, the API sweeps any row whose session_id is not in the active sessions table and deletes it. Documented behavior, not a leak.

3.3 detection/ — entity recognition

class Entity(BaseModel):
    type: str          # "PHI_DIAGNOSIS", "PII_SSN", etc.
    text: str
    start: int
    end: int
    confidence: float  # 0..1

class Detector(Protocol):
    async def detect(self, text: str) -> list[Entity]: ...

Presidio AnalyzerEngine wrapped in asyncio.to_thread. Built-in recognizers cover NAME/SSN/DOB/EMAIL/PHONE/ADDRESS. Custom recognizers for the rest:

Entity typeRecognizer
PHI_DIAGNOSISDictionary of common condition terms + ICD-10-code regex
PHI_MEDICATIONDictionary of drug names + dosage regex \d+\s?(mg|mL|mcg)
PHI_MRN\bMRN[:\s#]*\d{6,10}\b
PHI_INSURANCE_IDCommon insurer prefix patterns
FIN_ACCOUNT_NUMBERLuhn-validated 13–19 digit sequences
FIN_TAX_IDEIN regex + SSN format via built-in
LEGAL_PRIVILEGEHeader phrases: "ATTORNEY-CLIENT PRIVILEGED", "WORK PRODUCT", "CONFIDENTIAL — LEGAL"

Custom recognizers live in detection/recognizers/ — each is a Presidio PatternRecognizer registered via one line in detection/__init__.py. Adding PASSPORT_NUMBER = one new file + one registration line, zero changes elsewhere. Meets PRD's extensibility NFR.

Confidence threshold: configurable per-entity-type (default 0.6). Entities below threshold → fully redacted as [REDACTED_{TYPE}] per PRD's graceful degradation NFR.

3.4 obfuscation/strategies/ — Strategy pattern

class ObfuscationStrategy(ABC):
    @abstractmethod
    async def replace(self, entity: Entity, vault: Vault) -> str:
        """Return the replacement string and register it in the vault."""

class TokenizationStrategy(ObfuscationStrategy):
    """Produces [TYPE_HASH] opaque tokens. Vault maps token → original."""

class PseudonymizationStrategy(ObfuscationStrategy):
    """Produces realistic surrogates (Faker, type-aware). Vault maps surrogate → original."""

Shared abstract base class per PRD: "obfuscation strategies implement a shared base class — adding a new strategy touches zero harness code." Strategy selected at session start via OBFUSCATION_STRATEGY env var → meets "strategy swappability via config" NFR. Adding a new strategy = new file + one line in the factory. Zero changes to vault, detection, engine, or pipeline.

3.5 obfuscation/engine.py — orchestrator

class ObfuscationEngine:
    def __init__(self, detector: Detector, strategy: ObfuscationStrategy,
                 vault: Vault, audit: AuditLog): ...

    async def obfuscate(self, text: str) -> str:
        """Detect → for each entity, apply strategy → splice back into text."""

Spans replaced right-to-left so offsets stay valid. Emits one audit event per entity.

3.6 deobfuscation/ — response token restoration

PRD Core Req #6 mandates detecting "all token references including paraphrased / grammatically inflected forms." Two restoration paths, one per strategy:

Tokenization restoration

TOKEN_RE = re.compile(r"\[[A-Z]+(?:_[A-Z]+)+_[a-z2-7]{8}\]")
# Matches [PHI_NAME_k7a2mqpz], [PHI_INSURANCE_ID_p3hxk5n7], [LEGAL_PRIVILEGE_a4...].
# Survives trailing punctuation because the regex doesn't anchor to what follows:
# "[PHI_NAME_k7a2mqpz]'s" still extracts the bracketed token cleanly.

Each match → vault lookup → replace. Token not in vault (e.g. destroyed session) → emit audit event + leave [UNRESOLVED_TOKEN] in output.

Pseudonymization restoration

Surrogates ARE the form the LLM saw, and the LLM will inflect them like real words ("Torres's appointment," "the Torres family"). Restoration:

  1. Build a per-session reverse map {surrogate → original} from the vault.
  2. For each surrogate, compile a regex with \b word boundaries that tolerates trailing 's, s, punctuation.
  3. Apply longest-surrogate-first to avoid partial matches ("Michael Torres" before "Michael").
Hard post-condition: a final pass asserts no TOKEN_RE match remains in the user-facing output. If one does, the pipeline raises rather than leak a token. Maps directly to PRD's "no tokens leak to user output."

3.7 store/ — encrypted document storage

class DocumentStore(Protocol):
    async def put(self, user_id: str, doc_id: str,
                  content: bytes, *, filename: str) -> None: ...
    async def get_text(self, user_id: str, doc_id: str) -> str: ...

Filesystem-backed under ./data/<user_id>/<doc_id>.enc. Each file is Fernet(user_key).encrypt(raw_bytes). User isolation enforced by key AND by path scoping. aiofiles for non-blocking I/O.

Supported formats: TXT, DOCX, PDF (PRD Core Req #1) plus PNG, JPG (added per architect — scanned intake forms / lab results are a primary real-world input). Parser dispatched by filename suffix on get_text:

  • .txt → decode utf-8
  • .docx → python-docx
  • .pdf → pypdf; if text extraction yields empty/sparse output, fall back to per-page rasterize (pdf2image) + OCR
  • .png / .jpg / .jpeg → Tesseract OCR via pytesseract
OCR must be local. Cloud OCR services (AWS Textract, Google Vision, Anthropic Vision) all require sending the unredacted image to an external service — that violates threat model A1 (external provider must never see raw PII). Tesseract runs entirely in-process; no PII leaves the box. Documented limitation: Tesseract is strong on printed/typed text, weak on handwriting — README will note this honestly.

System dependencies (added to docker-compose api service): tesseract-ocr, poppler-utils (for pdf2image). Parsing is sync; wrapped in asyncio.to_thread to keep the pipeline non-blocking.

3.8 pipeline/ — end-to-end orchestrator

class Pipeline:
    async def run(self, user_id: str, doc_id: str, user_query: str) -> PipelineResult:
        """Load doc → obfuscate → call LLM → restore → return."""

class PipelineResult(BaseModel):
    detected_entities: list[Entity]      # for UI display
    obfuscated_text: str                 # what was sent to the LLM
    llm_response_raw: str                # response containing tokens/surrogates
    restored_text: str                   # final user-facing text
    audit_event_ids: list[str]           # for traceability in UI

Returning the intermediate stages lets the UI show the full pipeline visually. None of the intermediates contain raw PII outside the local process boundary — the request never leaves the user's session.

Session lifecycle: start_session(user_id) → generates SVK, creates vault. pipeline.run(...) uses that session's vault. end_session() → vault.destroy() + drops SVK from memory.

3.9 LLM client — Anthropic, async

Thin wrapper around anthropic.AsyncAnthropic. Default model claude-sonnet-4-6, configurable via env. System prompt asserts that bracketed tokens represent redacted entities and must be preserved verbatim in the response.

4. UI, API & Observability

Not in PRD core requirements but needed to demo the pipeline visually.

4.1 api/ — FastAPI HTTP layer

Thin async wrapper around the pipeline. Holds session state in-process (single-instance; HA is out of scope per §7).

EndpointPurpose
POST /sessionsBody: {user_id, strategy}. Creates session + vault, returns session_id.
DELETE /sessions/{id}Destroys vault, drops SVK. Audit-logged.
POST /sessions/{id}/documentsMultipart upload (txt/docx/pdf). Returns doc_id.
POST /sessions/{id}/pipelineBody: {doc_id, user_query}. Returns PipelineResult with all 4 stages for UI display.
GET /sessions/{id}/auditList audit events for this session (tokens + types only, never values).

A "vault stats" endpoint (counts by entity type) was considered and dropped: combined with the obfuscated payload visible in panel 3, exposing aggregate counts creates a confirm-by-presence side channel. The UI counts entity types directly from the tokens already present in the obfuscated text.

CORS enabled for the UI origin only. Each handler is async; errors map to typed problem-detail JSON.

4.2 ui/ — React + TypeScript + Vite

Single-page demo with two view modes:

  • Dev mode (default ON): the four-panel pipeline visualization — source → detected entities → obfuscated payload → LLM response → restored. This is the whole point of the demo: making the security properties visible to a reviewer.
  • User mode (?dev=false query string): clean chat-style view — just upload, query input, and the restored response. Demonstrates that the production UX hides the pipeline machinery entirely.

Same API and same backend behavior in both modes — the toggle only affects rendering. The API always returns the full PipelineResult; the UI decides what to render.

No router, no Zustand — TanStack Query alone manages server state; React's useState handles ephemeral UI state (current document, strategy toggle, dev/user mode). Per the "YAGNI until the third occurrence" principle.

What the UI proves to a reviewer

  1. Detection works — panel 2 lists detected spans with type + confidence.
  2. Obfuscation works — panel 3 shows the exact bytes sent to the LLM, manually inspectable for any leaked PII.
  3. Restoration works — panel 4 toggle proves the LLM only saw tokens but the user sees originals.
  4. Audit purity — live audit log in sidebar contains only token IDs and types.
  5. Session isolation — "End session" wipes the vault; uploading the same doc in a new session produces different tokens.

4.3 Observability — optional LangSmith integration

Independent verification of "zero PII in transit" beyond our own panel 3. When LANGSMITH_TRACING=true and LANGSMITH_API_KEY are set in env, the Anthropic async client is wrapped to emit traces to LangSmith. A reviewer can open the LangSmith UI and inspect the exact request/response bytes the provider saw — and confirm independently that no original PII appears.

Implementation: thin wrapper in llm_client/tracing.py. If tracing is disabled or the API key is unset, the wrapper is a no-op (no SDK import, no overhead). Trade-off documented: enabling LangSmith routes (already-obfuscated) prompts through a third-party service — fine for verification of the security guarantee, but disclosed in README.

5. Build Order

Sequenced for clean dependency flow — each step compiles & tests cleanly before the next:

  1. Project scaffold — pyproject.toml, directory tree, .env.example, docker-compose.yml (api + ui), pytest config, ruff/mypy config.
  2. audit/ — pydantic event model + JSONL writer + tests asserting no-value-leakage.
  3. vault/ — SQLite schema, HMAC token derivation, Fernet row encryption, session lifecycle, cross-session isolation tests.
  4. detection/ — Presidio wrapper, custom recognizers, confidence threshold, tests on fixture text.
  5. obfuscation/strategies/ — Tokenization + Pseudonymization implementing the ABC.
  6. obfuscation/engine.py — splicing logic, right-to-left replacement, audit emission.
  7. store/ — encrypted doc store + TXT/PDF/DOCX extraction.
  8. LLM client — async Anthropic wrapper + echo-mock for tests.
  9. deobfuscation/ — token regex + vault lookup + restoration for both strategies.
  10. pipeline/ — session manager + end-to-end orchestrator returning PipelineResult.
  11. api/ — FastAPI handlers wrapping the pipeline.
  12. ui/ — Vite/React app with the four-panel layout.
  13. demo.py — CLI alternative to the UI for headless demo. Runs the full flow on a bundled fixture.
  14. Integration tests — all PRD success criteria automated.
  15. README — threat model, design rationale, tokenization vs pseudonymization comparison, known gaps, "next day" list.

Each step has unit tests written alongside the code, not after. We do not move to step N+1 until step N's tests are green and mypy/ruff are clean.

7. What We Are NOT Building (scope guard)

Out of MVP. Any of these requires a fresh decision from the architect:

  • User authentication — user_id is a string passed into the API. PRD does not require auth.
  • Cloud storage (S3, MinIO, GCS) — PRD labels optional; local filesystem only.
  • KMS integration (AWS, GCP) — PRD allows mocks; env-var keys.
  • SQLCipher — PRD allows "SQLite + SQLCipher OR encrypted file store." Per-row Fernet on plain SQLite satisfies "vault entries encrypted" with fewer deps.
  • LLM-based entity detection — PRD allows it; we picked Presidio. No fallback hop.
  • Co-reference resolution — README known gap.
  • Handwriting OCR — Tesseract handles printed/typed text only. Documents with handwritten fields are not in scope.
  • Multi-language detection — Presidio English-only.
  • High-availability vault (replication, multi-instance) — single SQLite file is the MVP.
  • Auth on the UI — single-user demo, no login screen.
  • Persistent UI state — refreshing the page can reset the session view; the vault stays alive server-side until DELETE /sessions/{id}.

8. Known Gaps (will be documented in README per PRD §README requirements)

  • Co-reference — LLM-produced indirect references ("the patient," "this individual") aren't restored because they were never tokenized. Live-review Q2 territory.
  • Frequency analysis on pseudonymized output — adversary with both obfuscated doc and response can count surrogate occurrences and infer entity importance. Live-review Q3 territory.
  • Vault key in memory — SVK is in process memory; an attacker with memory-read access defeats the scheme. Production needs HSM or KMS-resident keys.
  • Pseudonymization inflection edge cases — the per-surrogate regex handles trailing 's / s / punctuation, but pluralization that mutates the surrogate ("the Torreses"), lowercased mid-sentence ("torres"), or possessive-without-s ("Torres'") will not restore. Tokenization is unaffected.
  • Vault lookup timing side-channel — hit vs miss timing on SQLite is measurable; an attacker with API access could probe for token existence. Production: constant-time lookup or rate limiting.
  • Handwriting OCR, multi-language, vault HA — see §7.
