4. Cross-session isolation, manually — strongest demo of the cryptographic property.

  • End session A.
  • Start session B (same user_id, same strategy), upload the same fixture, run pipeline.
  • The token for "Sofia M. Reyes" should be different between the two runs.

5. Edge cases worth probing

  • Very large doc (paste an 8000-word fixture) — does perf hold up? (PRD budget: 15s end-to-end for 2k words.)
  • Doc with no PII (just generic text) — pipeline should still work, return original text unchanged.
  • Doc with mixed languages — Presidio is English-only; non-English entities will leak. Known gap, but worth confirming the behavior matches expectations.
  • Restart the API process between sessions — orphan-cleanup should sweep stale vault rows.

If something breaks during manual testing

  • The next session should start by reading this file, PLAN.html, and README.md. The full design rationale and known gaps are documented.
  • Specific bugs to watch for (none observed in automated tests, but unverified in real-world flow):
    • Anthropic might paraphrase tokens despite the system prompt — if restoration looks incomplete, check whether the LLM modified the bracketed strings.

    • Pseudonymization inflection on uncommon surrogates (foreign names, multi-word) — documented in PLAN §8 gaps; if restoration leaves a surrogate behind, that's a known issue.
